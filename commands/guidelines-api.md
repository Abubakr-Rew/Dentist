# API & Data Management Guidelines

## 1. Режим автономной работы (Vibe Coding Mode / Mocks)
- На этапе верстки и дизайна фронтенд должен полноценно работать в автономном режиме без поднятого Бэкенда.
- Для этого используются Mock-данные (массивы объектов в `src/mocks/data.ts`).
- Всегда нужно предусматривать быстрое переключение между `mock`-данными и вызовами к API.
- На страницах (например, `Home.tsx` или списках) временно импортируются `mockClinics`, пока бэкенд не запущен.

## 2. Структура реального API
- Вызовы к реальному бэкенду лежат в `src/services/api.ts` (или разделены на коллекции: `clinicsApi`, `authApi`).
- Все вызовы API проксируются через `/api/*` (на порт `localhost:4000`), чтобы избежать проблем с CORS на этапе локальной разработки. Это настроено в `vite.config.js`.
- При полномасштабном запуске (вместе с базой PostgreSQL и Express.js) компоненты должны запрашивать данные через `await clinicsApi.list()`.

## 3. Обработка ошибок (Error Handling)
- API запросы должны быть обернуты в `try/catch`. 
- При ошибке сервера (500/504) или при выключенном бэкенде система должна показывать адекватные ошибки на фронтенде (сообщения Toast, уведомления), а не просто ломать интерфейс пользователя.
