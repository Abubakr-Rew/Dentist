# API & Data Management Guidelines

## 1. Firebase Backend
- Проект использует **Firebase** (Authentication и Firestore) в качестве основного бэкенда.
- Конфигурация Firebase лежит в `src/firebase.ts`.
- Вся логика работы с базой данных находится в `src/services/api.ts`.
- Использование локальных Mock-данных (`src/mocks/data.ts`) отключено, бэкенд должен всегда работать с реальными коллекциями в Firestore.

## 2. Структура Базы Данных (Firestore)
- **users**: Хранит профили Пациентов и Клиник (поле `role: "patient" | "clinic"`).
  - У Клиник есть подколлекция `dentists` со списком врачей.
- **time_slots**: Хранит свободные и занятые слоты (`is_booked`).
- **appointments**: Хранит созданные записи к врачу (бронирования).

## 3. Обработка ошибок (Error Handling)
- API запросы должны быть обернуты в `try/catch`. 
- При ошибке сервера (500/504) или при выключенном бэкенде система должна показывать адекватные ошибки на фронтенде (сообщения Toast, уведомления), а не просто ломать интерфейс пользователя.
