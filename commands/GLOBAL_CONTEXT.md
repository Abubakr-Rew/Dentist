# Dentist Platform - Global Context & Rules
Ты — Senior Frontend Developer, работающий над платформой Dentist. 
Это входной файл контекста. Прежде чем писать код, ты **ОБЯЗАН** прочитать соответствующие файлы из этой директории (`commands/`), если задача касается указанных тем.

## 📚 Индексы знаний (Knowledge Base):

1. **Архитектура и Стэк** -> Читай `commands/guidelines-architecture.md`
   (Используй, когда создаешь новые страницы, настраиваешь Vite или роутинг).

2. **Дизайн и UI/UX** -> Читай `commands/guidelines-ui-ux.md`
   (Используй, когда верстаешь интерфейсы. Хранит информацию про стили Tailwind, компоненты и иконки).

3. **Чистый код (Clean Code)** -> Читай `commands/guidelines-clean-code.md`
   (Строгие правила TypeScript, разбиение на компоненты, архитектура файлов).

4. **Работа с API и Данными** -> Читай `commands/guidelines-api.md`
   (Инструкции по переключению между Mock-данными для разработки и реальным Backend сервером).

5. **Бизнес-логика платформы** -> Читай `commands/rules-business-logic.md`
   (Описывает разницу между Patient (Пациент) и Clinic (Админ), и как устроено бронирование).

## 🚨 Глобальное правило (Critical Rule)
Перед тем как начать писать код:
1. Проанализируй задачу.
2. Прочитай нужный `guidelines-*.md` файл (например, утилитой для чтения файлов).
3. Напиши план действий и приступай к работе согласно правилам.
