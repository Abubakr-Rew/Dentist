# UI/UX & Styling Guidelines

## 1. General Vibe
- **Стиль:** Современный, чистый, вызывающий доверие медицинский UI. 
- **Цветовая палитра:** Использование светлых тем, с акцентами из primary blues/teals. Свежий медицинский стиль. Красный для отмен/ошибок, зеленый для успешных действий/завершения. Серый для второстепенного текста.

## 2. Тайпографика и Tailwind
- Использовать ТОЛЬКО классы `Tailwind CSS`. Создавать кастомные стили через `.css` файлы ЗАПРЕЩЕНО, кроме базового `index.css`.
- Дизайн должен выглядеть "дорого" (премиально). Используйте скругления (`rounded-xl`, `rounded-2xl`), мягкие тени (`shadow-sm`, `shadow-lg`), микроанимации при наведении (`hover:scale-105`, `transition-all`).
- Использовать современные шрифты без засечек (Inter, Roboto).

## 3. Иконки
- **БИБЛИОТЕКА:** `@phosphor-icons/react`.
- **КРИТИЧНО:** Запрещено использовать `lucide-react` или любые другие библиотеки иконок. Эта ошибка часто ломает единый стиль проекта.

## 4. Адаптивность (Mobile-First)
- Приложение должно быть полностью адаптивным (Mobile-First подход).
- Использовать префиксы `sm:`, `md:`, `lg:` для изменения структуры сетки (например, вертикальные карточки на мобильных и таблицы `table` на десктопах).
- Меню и выпадающие списки не должны ломать верстку при переполнении (overflow). Абсолютно позиционированные элементы должны корректно перекрывать контент (`z-index`).

## 5. Обратная связь UI
- Обязательно показывать пустые состояния (Empty states), например: "Записей на эту дату нет" или "Клиники не найдены".
- Использовать skeleton loading или спиннеры при загрузке.
