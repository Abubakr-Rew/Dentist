# Business Logic & App Features (Dentist Platform)

Платформа "Dentist" — это сервис бронирования приемов (цифровая стоматологическая поликлиника). Приложение разделено на два основных независимых флоу.

## 1. Patient Flow (Пациент)
- **Целевая аудитория:** Обычные пользователи платформы.
- **Главная и поиск:** Пациенты могут просматривать список всех зарегистрированных клиник платформы, фильтровать их по услугам, находить нужного врача.
- **Запись на прием (Booking):** Процесс создания объекта `Appointment` (Бронирования). Пациент выбирает Клинику -> Врача -> Доступную Услугу -> Дату и Время.
- **Ограничения:** Визуально нужно блокировать на календаре слоты с датами и временем, которые уже заняты другими пользователями (система исключает double-booking).
- **Кабинет Пациента:** Защищенная зона (`ProtectedRoute`), где можно посмотреть свои предстоящие записи и историю визитов.

## 2. Clinic Admin Flow (Админ клиники / Ресепшен)
- **Целевая аудитория:** Администраторы стоматологии и врачи.
- **Дашборд (`ClinicDashboard.tsx`):** Главная страница для администратора. Содержит расписание клиники на день/неделю. Приложение должно позволять изменять статусы приемов (upcoming -> completed -> cancelled).
- **Управление профилем:** Администратор должен иметь возможность добавлять/удалять врачей и редактировать перечень услуг с ценами.

## 3. Scope MVP (Первый релиз)
- Фокус ИСКЛЮЧИТЕЛЬНО на расписаниях, флоу бронирования, поиске клиник и базовой аутентификации пользователя (`AuthContext`, JWT-токены в localStorage).
- **СТРОГО ЗАПРЕЩЕНО (Выходит за рамки MVP):** Интеграция платежных шлюзов (Stripe/Paypal), системы страховок, видеозвонков, телемедицины, AI-диагностики зубов по фото. Код под эти фичи генерировать не нужно на данном этапе проекта.
