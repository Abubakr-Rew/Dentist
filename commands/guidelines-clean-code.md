# Clean Code & React Guidelines

## 1. TypeScript Rules
- Strict mode обязателен.
- Использовать `any` СТРОГО ЗАПРЕЩЕНО. 
- Все Props компонентов должны строго описываться через `interface` или `type`.
- При импорте интерфейсов использовать импорты из папки `src/types` (если необходимо) или экспортировать типы рядом с моделями.

## 2. Разделение бизнес-логики и UI
- Большие компоненты необходимо разбивать. Если файл превышает 150-200 строк, логику выносить в отдельные компоненты или кастомные хуки (например, `useAppointments.ts`).
- Принцип единой ответственности: Страница (Page) занимается стейтом и агрегацией данных, UI компонент (Card, Table) занимается только отображением данных.

## 3. Импорты и кодстайл
- Порядок импортов:
  1. Built-in React хуки (`react`).
  2. Внешние зависимости (например, `@phosphor-icons/react`).
  3. Локальные UI-компоненты (`../components/ui`).
  4. Локальные утилиты и типы, мок-данные.
- Стараться не дублировать код. Если верстка (например, кнопка или форма) используется больше 2-х раз, категорически рекомендуется вынести ее в компонент `components/ui`.
- Избегать inline функций в JSX там, где это вызывает лишние рендеры в тяжелых компонентах (по возможности оборачивать в `useCallback`).
